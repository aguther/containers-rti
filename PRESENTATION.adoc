== Vagrant

[%notitle]
=== Vagrant
[quote,wikipedia.org]
Vagrant is an open-source software product for building and maintaining portable virtual development environments.

=== How to use?

=== Installation
[%step]
* Windows, Linux or macOS
* Administrative Account
* VirtualBox
* Vagrant

=== Get started
[source,bash]
----
$ vagrant init bento/centos-7.3
$ vagrant up
----

=== Multi-Machine
[source,ruby]
----
Vagrant.configure("2") do |config|

  config.vm.define "web" do |web|
    web.vm.box = "apache"
  end

  config.vm.define "db" do |db|
    db.vm.box = "mysql"
  end

end
----

== Ansible

[%notitle]
=== Ansible
[quote,wikipedia.org]
Ansible is an open-source automation engine that automates cloud provisioning, configuration management, and application deployment.

[%notitle]
=== Ansible
[quote,ansible.com]
____
*AUTOMATION FOR EVERYONE*

Deploy apps. Manage systems. Crush complexity.
Ansible helps you build a strong foundation for DevOps.
____

=== How to use?

=== Installation
* RHEL / CentOS with EPEL repository +
  `yum install ansible`
* Other platforms +
  see http://docs.ansible.com/ansible/intro_installation.html[docs.ansible.com]

=== Get started
[%step]
. Create inventory
. Execute desired command

=== Inventory Creation
[source,bash]
----
$ touch ./hosts
$ vim ./hosts
----

=== Inventory Content
[source,ini]
----
[centos]
centos-1
centos-2
centos-3

[centos:vars]
ansible_user=vagrant
ansible_ssh_password=vagrant
----

=== Execute Command
[source,bash]
----
$ ansible -i ./hosts all -a "hostname"
----

== Docker

=== Dockerfile
[source,docker]
----
# define parent
FROM centos:7

# add RTI Perftest binary package
ADD https://[...] /rtiperftest.tar.gz

# extract archive
RUN tar xzf /rtiperftest.tar.gz

# define work directory and entrypoint
WORKDIR "[...]"
ENTRYPOINT ["[...]/perftest_cpp"]
----
